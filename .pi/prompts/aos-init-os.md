---
description: Redirigir inicializaciones AOS al upstream manager
---
Este repo no es el manager para inicializar otros proyectos. Si JP pide `aos-init-os` para otro repo, no lo ejecutes desde aca: indica usar `C:/dev/os` como upstream manager y conservar este repo como downstream local. Si el objetivo real es mejorar la capa AOS de `main-v2`, usa `aos-realinear-os`/`aos-perfect-os`.
